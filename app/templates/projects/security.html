API Access
  • All API endpoints require Bearer token authentication
  • Tokens are scoped to specific projects and environments
  • IP whitelist applies to all API requests
  • Failed authentication attempts are logged
Data Protection
  • Secrets are encrypted using environment-specific keys
  • Encryption uses Fernet (AES 128 in CBC mode)
  • Keys are generated cryptographically secure
  • Configurations are stored in plain text for performance
Best Practices:
  • Use specific IP addresses or small ranges instead of broad networks
  • Use FQDNs for dynamic IP addresses (e.g., api.example.com)
  • Regularly review and remove unused IP addresses and FQDNs
  • Set API token expiration dates appropriately
  • Monitor access logs for suspicious activity
